Brand Gaps

What is still unresolved

The visual language is already strong enough to build from. What is missing is a decision layer that turns these related parts into one governable system.

No single primary logo standard

Three marks are active across local assets and live usage, but there is no explicit hierarchy showing which one leads on web, print, social, and merchandise.

Palette alignment is incomplete

The website is disciplined around cocoa and gold, while the supplied logos lean richer, warmer, and more illustrative. Both feel related, but they are not yet normalized.

Button semantics are muddy

Bootstrap classes have been visually remapped, which makes the UI look branded but weakens meaning and maintainability. The color-role system needs a clean reset.

Core web identity assets are incomplete

The live site still uses a default favicon and default share image, so key brand touchpoints outside the page chrome remain generic.

Responsive logo placement is brittle

The header uses many breakpoint-specific spacing adjustments, which suggests the web logo rules have not been simplified into a stable system.

Typography relationship is not documented

Poppins dominates the site, while the strongest logo mark carries a more traditional serif feel. That can be a strength, but only if the relationship is intentional.
